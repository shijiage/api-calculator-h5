## 1. 背景与问题

### 1.1 问题陈述

用户在选择 API 中转站时，面临两个核心困境：

1. **信息不对称**：官方推荐站点数据由维护者人工整理，更新频率有限，无法反映真实用户的近期使用体验。
2. **决策孤立**：用户找到好站点后没有渠道分享，好的信息沉淀在个人手里，社区整体效率低。

### 1.2 机会

产品已有一定用户基础，用户在使用对比报告时会形成对站点质量的真实判断。这些判断目前「用完即走」，没有沉淀成可被他人复用的内容。

### 1.3 目标

让用户把自己用过的好站点分享给社区，通过点赞和评论形成有温度的互动，让真正有价值的推荐通过火热度排序浮出水面。

## 2. 用户故事

| 编号 | 用户角色 | 故事 | 验收条件 |
|------|----------|------|----------|
| US-01 | 已使用对比功能的用户 | 我对比完两个站点、发现 A 站性价比更高，我想把 A 站推荐给其他用户 | 报告页有推荐入口，点击后能完成推荐 |
| US-02 | 普通访客 | 我想看看其他用户推荐了哪些站点 | 发现页有社区推荐分区，无需登录可浏览 |
| US-03 | 已登录用户 | 我想手动推荐一个好站点，不一定来自报告 | 发现页有手动推荐入口 |
| US-04 | 已推荐过的用户 | 我推荐了一个站点后发现填错了，我想删掉重来 | ~~不支持删除，推荐一经发布不可撤回~~ |
| US-05 | 普通访客 | 我想在社区分享区集中浏览用户推荐的站点 | 发现页有独立的「大家的推荐」分享区，与官方推荐区完全分开 |
| US-06 | 浏览用户 | 我看到一个好推荐，想点个赞表示认可 | 推荐卡片有点赞按钮，已登录可点赞，点赞数实时更新 |
| US-07 | 浏览用户 | 我想就某个推荐补充一些使用体验 | 推荐卡片有评论入口，已登录可发评论 |
| US-08 | 浏览用户 | 我不知道从哪条推荐开始看，想优先看最受欢迎的 | 分享区支持「火热」排序，默认按热度展示 |

## 3. 功能范围

### 3.1 核心功能（本期）

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 推荐内容填写 | P0 | 站点名称 + 站点链接 + 一句话评价，三项必填 |
| 从报告一键推荐 | P0 | 报告页点击推荐按钮，打开填写弹窗，三项均需用户手动填写 |
| 发现页手动推荐 | P0 | 不依赖报告，直接在发现页推荐 |
| 社区推荐列表 | P0 | 发现页独立的「大家的推荐」分享区，与官方推荐数据完全隔离 |
| 点赞功能 | P0 | 登录后可对推荐卡片点赞 / 取消点赞，显示点赞数 |
| 评论功能 | P0 | 登录后可在推荐卡片下发评论，显示评论数 |
| 火热度排序 | P0 | 根据点赞数和评论数综合计算热度，作为默认排序方式 |
| 最新排序 | P1 | 支持切换为按发布时间倒序排列 |
| 官方卡片下方展示评价 | ~~移除~~ | 用户推荐内容不进入官方推荐区，两块数据完全分开 |

### 3.2 不在本期范围

- 内容举报与审核
- 站点评分（星级）
- 用户个人推荐历史页面
- 推荐内容的搜索与筛选
- 评论的点赞 / 回复（评论只支持一级，不嵌套）

## 4. 用户流程

### 4.1 流程一：从报告推荐（主路径）

```
用户生成对比报告
    │
    ▼
报告页看到「推荐这个站点」按钮（A站 / B站各一个）
    │
    ├── 未登录 ──→ 弹出登录引导 ──→ 登录后回到报告页
    │
    └── 已登录
          │
          ▼
        弹出填写弹窗（站点名称、站点链接、一句话评价均需手动填写）
          │
          └── 点击「提交推荐」
                │
                ├── 校验不通过 ──→ 行内提示错误
                │
                └── 校验通过
                      │
                      ▼
                    提交成功 ──→ 弹窗关闭，Toast 提示「推荐成功」
```

### 4.2 流程二：发现页手动推荐

```
发现页 → 推荐站点 Tab
    │
    ▼
点击「+ 推荐站点」按钮
    │
    ├── 未登录 ──→ 弹出登录引导
    │
    └── 已登录
          │
          ▼
        弹出填写弹窗（三项均需手动填写）
          │
          └── （同流程一后续步骤）
```

### 4.3 流程三：浏览、点赞、评论

```
发现页 → 推荐站点 Tab → 大家的推荐分区
    │
    ├── 默认按火热度排序展示推荐卡片
    │
    ├── 点赞
    │     ├── 未登录 ──→ 弹出登录引导
    │     └── 已登录
    │           ├── 未点赞 ──→ 点击点赞图标 ──→ 点赞数 +1，图标高亮
    │           └── 已点赞 ──→ 点击点赞图标 ──→ 取消点赞，点赞数 -1，图标恢复
    │
    └── 评论
          ├── 未登录 ──→ 点击评论图标 ──→ 弹出登录引导
          └── 已登录
                ├── 点击评论图标 ──→ 展开评论区（卡片内展开）
                │     └── 显示已有评论列表 + 底部输入框
                └── 输入评论内容 ──→ 点击「发送」
                      ├── 内容为空 ──→ 不可发送
                      └── 发送成功 ──→ 评论数 +1，新评论出现在列表顶部
```

## 5. 页面与交互设计

### 5.1 填写弹窗（两个入口共用）

```
┌─────────────────────────────────┐
│  推荐一个好站点            [✕]  │
├─────────────────────────────────┤
│                                 │
│  站点名称 *                     │
│  ┌─────────────────────────┐   │
│  │                         │   │
│  └─────────────────────────┘   │
│                                 │
│  站点链接 *                     │
│  ┌─────────────────────────┐   │
│  │ https://                │   │
│  └─────────────────────────┘   │
│                                 │
│  一句话评价 *          0 / 50   │
│  ┌─────────────────────────┐   │
│  │                         │   │
│  └─────────────────────────┘   │
│                                 │
│  ┌─────────────────────────┐   │
│  │       提交推荐           │   │
│  └─────────────────────────┘   │
│                                 │
└─────────────────────────────────┘
```

**交互细节：**
- 两个入口（报告页 / 发现页）打开的是同一个弹窗，三项均需用户手动填写
- 当前对比工具不记录站点名称，用户需要自行填写以便社区其他用户识别
- 评价实时显示字数，超过 50 字时输入框边框变红
- 提交中状态：按钮文案改为「提交中…」并禁用
- 链接不含 `http://` 或 `https://` 时，下方显示「请输入完整链接」

### 5.2 发现页「大家的推荐」分享区

用户推荐内容**只在此区展示**，与官方推荐区完全隔离，两块数据互不干扰。

```
┌─── 推荐站点 Tab ───────────────────────────────┐
│                                                 │
│  ══ 官方推荐 ═══════════════════════════════    │
│  [官方推荐内容（现有，不变）]                   │
│                                                 │
│  ══ 大家的推荐 ══════════════════════════════   │
│  [🔥 火热] [最新]                [+ 推荐站点]   │
│                                                 │
│  ┌── 推荐卡片 ───────────────────────────────┐  │
│  │ 某中转站                      2小时前      │  │
│  │ 「延迟稳定，价格比官方便宜30%，强推」       │  │
│  │ 链接：https://xxx.com  [复制]             │  │
│  │ by abc123***                              │  │
│  │ ────────────────────────────────────      │  │
│  │ 👍 24        💬 6                          │  │
│  └───────────────────────────────────────────┘  │
│                                                 │
│  ┌── 推荐卡片（展开评论状态）────────────────┐  │
│  │ 另一个站点                    昨天         │  │
│  │ 「新用户有折扣，响应速度不错」              │  │
│  │ 链接：https://yyy.com  [复制]             │  │
│  │ by def456***                              │  │
│  │ ────────────────────────────────────      │  │
│  │ 👍 8         💬 3  ▲                      │  │
│  │ ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄  │  │
│  │ ghi789***：用了两周很稳定        1天前     │  │
│  │ jkl012***：新用户有额外折扣      2天前     │  │
│  │ ┌────────────────────────────┐           │  │
│  │ │ 说点什么…                  [发送]      │  │
│  │ └────────────────────────────┘           │  │
│  └───────────────────────────────────────────┘  │
│                                                 │
└─────────────────────────────────────────────────┘
```

**交互细节：**

**排序：**
- 默认「🔥 火热」排序，高亮显示当前排序
- 「最新」切换为按发布时间倒序
- 切换排序时列表重新加载

**点赞：**
- 点赞图标默认灰色，点赞后变为高亮（蓝色或橙色）
- 同一用户对同一推荐只能点赞一次，再次点击为取消点赞
- 未登录点击 → 弹出登录引导

**评论：**
- 点击评论图标（💬）展开 / 收起评论区，同一时间只展开一张卡片的评论
- 评论展开后显示已有评论列表（最多显示最新 10 条）和底部输入框
- 评论内容限 100 字，发送后出现在列表顶部
- 未登录点击 → 弹出登录引导
- 评论只有一级，不支持回复嵌套

**其他：**
- 「+ 推荐站点」按钮：未登录触发登录引导；已登录直接打开弹窗
- 分享区为空时显示：「还没有人推荐，来第一个分享好站点」
- 推荐一经发布不可撤回

### 5.3 报告页推荐入口

在报告页「分享报告」按钮上方新增操作区：

```
┌── 推荐到社区 ─────────────────────────────────┐
│                                                │
│  觉得这个站点不错？推荐给其他用户              │
│                                                │
│  [推荐 A 站]          [推荐 B 站]              │
│                                                │
└────────────────────────────────────────────────┘
```

**交互细节：**
- 两个按钮分别对应 A 站和 B 站（按钮上不显示站点名，因工具不记录站点名）
- 点击后打开填写弹窗，三项均需用户手动填写

### 5.4 ~~官方站点卡片下方评价~~（已移除）

用户推荐内容只在「大家的推荐」分享区展示，不渗透到官方推荐区。官方站点卡片保持原样。

## 6. 火热度算法

### 6.1 计算公式

```
热度分 = 点赞数 × 2 + 评论数 × 3
```

**权重说明：**
- 评论权重高于点赞，因为发评论的意愿门槛更高，代表更强的用户认可
- 权重可后续根据数据表现调整，无需改动产品逻辑

### 6.2 排序规则

- 热度分相同时，按发布时间倒序作为次级排序
- 热度分在每次请求列表时由后端实时计算，不持久化存储

## 7. 内容规则

| 规则 | 说明 |
|------|------|
| 登录门控 | 必须登录才能发布推荐、点赞、评论；未登录只能浏览 |
| 推荐必填 | 站点名称、站点链接、一句话评价三项缺一不可 |
| 链接格式 | 必须以 `http://` 或 `https://` 开头 |
| 推荐评价长度 | 最多 50 字 |
| 评论长度 | 最多 100 字 |
| 防重复推荐 | 同一用户对同一站点名，每天最多推荐 1 次 |
| 点赞唯一 | 同一用户对同一推荐只能点赞一次，可取消 |
| 内容呈现 | 推荐人 / 评论人显示 openid 前 6 位 + `***` 脱敏 |
| 不可撤回 | 推荐和评论一经发布不支持用户自行删除 |

## 8. 成功指标

| 指标 | 定义 | MVP 目标 |
|------|------|----------|
| 推荐提交率 | 点击推荐按钮 → 成功提交的比例 | ≥ 60% |
| 社区分享区 DAU | 每天浏览「大家的推荐」的用户数 | ≥ 30% MAU |
| 推荐内容量 | 上线 30 天内累计推荐条数 | ≥ 50 条 |
| 从报告推荐转化率 | 生成报告用户中点击过推荐按钮的比例 | ≥ 10% |
| 互动率 | 有过点赞或评论行为的用户 / 浏览分享区用户 | ≥ 20% |
| 人均互动次数 | 有互动行为用户的月均点赞 + 评论次数 | ≥ 3 |
| 链接复制次数 | 社区推荐卡片的链接复制行为次数 | — |
